# PANDUAN ONBOARDING ADMIN EVENT SIAP PMI

## Tujuan
Panduan ini memastikan Admin Event memahami tugas operasional berbasis event, waktu pelaksanaan, dan artefak yang wajib dihasilkan pada sistem internal single province dengan role per event.

## Ruang Lingkup
Berlaku untuk operasional event pelatihan internal PMI pada satu provinsi, tanpa perubahan fitur, tanpa perubahan alur bisnis, dan mengikuti kebijakan change-freeze.

## Peran dan Tanggung Jawab Utama
- Menyiapkan event, jadwal, dan data inti sesuai kebutuhan kegiatan.
- Menetapkan role per event untuk Admin Event, Pelatih, Observer, dan Peserta.
- Memastikan enrollment, assessment, dan evaluation berjalan sesuai jadwal.
- Mengelola rapat kelulusan dan pencatatan keputusan manual.
- Menutup event setelah seluruh persyaratan operasional terpenuhi.
- Mengarsipkan dokumen operasional sebagai evidence internal.

## Alur Kerja Admin Event (Pra-Event sampai Penutupan)
### 1) Pra-Event
- Kapan: sebelum event dipublikasikan.
- Tindakan: siapkan data event, jadwal, kuota, dan penetapan role per event.
- Artefak: checklist pra-event, daftar role per event, notifikasi internal jadwal.

### 2) Enrollment Peserta
- Kapan: setelah event published sampai penutupan pendaftaran.
- Tindakan: verifikasi peserta sesuai kuota dan event yang benar.
- Artefak: daftar peserta terverifikasi, catatan validasi enrollment.

### 3) Pelaksanaan Event
- Kapan: hari pelaksanaan sesuai jadwal.
- Tindakan: pastikan akses sesuai role per event dan sesi berjalan tertib.
- Artefak: catatan pelaksanaan harian, bukti kehadiran atau catatan sesi.

### 4) Assessment Pelatih/Observer
- Kapan: sesuai jadwal assessment pada event.
- Tindakan: pastikan pelatih/observer mengisi assessment pada event yang tepat.
- Artefak: rekap kelengkapan assessment per sesi atau per peserta.

### 5) Evaluation Peserta
- Kapan: setelah sesi pelatihan sesuai jadwal evaluation.
- Tindakan: pastikan peserta mengisi evaluation pada event terkait.
- Artefak: rekap kelengkapan evaluation peserta.

### 6) Rapat Kelulusan
- Kapan: setelah assessment dan evaluation lengkap.
- Tindakan: fasilitasi rapat kelulusan; keputusan ditetapkan manual.
- Artefak: notulen rapat kelulusan, daftar keputusan peserta.

### 7) Penutupan dan Arsip
- Kapan: setelah seluruh data lengkap dan keputusan kelulusan tercatat.
- Tindakan: tutup event dan pastikan arsip dokumentasi lengkap.
- Artefak: rekap event, status penutupan event, paket arsip operasional.

## Checklist Harian Admin Event
- Validasi status event aktif dan jadwal kegiatan hari ini.
- Verifikasi role per event untuk pelatih, observer, dan peserta.
- Konfirmasi kesiapan sesi dan daftar hadir peserta.
- Pantau kelengkapan assessment/evaluation sesuai jadwal.
- Catat isu operasional dan tindak lanjut tanpa perubahan fitur.
- Simpan bukti komunikasi dan catatan operasional harian.

## Kesalahan Umum yang Harus Dihindari
- Menetapkan role secara global, bukan per event.
- Membiarkan peserta/enrollment tercatat pada event yang tidak tepat.
- Menganggap evaluation menentukan kelulusan secara otomatis.
- Menutup event sebelum assessment, evaluation, dan keputusan kelulusan lengkap.
- Melakukan perubahan data lintas event atau lintas provinsi.
- Melakukan perubahan di luar mekanisme change control saat change-freeze aktif.
