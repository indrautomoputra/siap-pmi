# PANDUAN ONBOARDING ADMIN EVENT SIAP PMI

## Pendahuluan
Admin Event adalah penggerak utama operasional SIAP PMI yang memastikan seluruh kegiatan event berjalan tertib, sesuai jadwal, dan patuh pada ketentuan internal. Peran ini memastikan setiap proses tetap berbasis event, role ditetapkan per event, dan seluruh pencatatan berjalan sesuai prosedur tanpa perubahan alur bisnis.

## Ruang Lingkup
Berlaku untuk operasional event pelatihan internal PMI pada satu provinsi, tanpa perubahan fitur, tanpa perubahan alur bisnis, dan mengikuti kebijakan change-freeze.

## Tanggung Jawab Utama Admin Event
Admin Event bertanggung jawab menyiapkan event dan jadwal, menetapkan role per event untuk pelatih, observer, dan peserta, serta memastikan enrollment, assessment, dan evaluation berjalan sesuai ketentuan. Admin Event juga memfasilitasi rapat kelulusan sebagai proses keputusan manual, menutup event setelah seluruh persyaratan lengkap, dan mengarsipkan dokumen operasional sebagai bukti audit internal.

## Alur Kerja Admin Event
### Pra-Event
Sebelum event dipublikasikan, Admin Event menyiapkan data event, jadwal, kuota, dan penetapan role per event. Pada tahap ini, Admin Event menyiapkan checklist pra-event, daftar role per event, serta notifikasi internal jadwal sebagai artefak operasional awal.

### Selama Event
Selama event berlangsung, Admin Event memastikan akses berjalan sesuai role per event, memverifikasi kehadiran dan kesiapan sesi, serta memantau pengisian assessment oleh pelatih/observer dan evaluation oleh peserta sesuai jadwal. Artefak yang dihasilkan berupa catatan pelaksanaan harian, bukti kehadiran, serta rekap kelengkapan assessment dan evaluation.

### Pasca-Event
Setelah assessment dan evaluation lengkap, Admin Event memfasilitasi rapat kelulusan dan memastikan keputusan kelulusan dicatat manual pada event terkait. Event ditutup setelah persyaratan lengkap, dan rekap event, status penutupan, serta paket arsip operasional disusun untuk kebutuhan audit.

## Checklist Harian Admin Event
Setiap hari, Admin Event memvalidasi status event aktif dan jadwal kegiatan, memverifikasi role per event untuk pelatih, observer, dan peserta, serta mengonfirmasi kesiapan sesi dan daftar hadir. Admin Event juga memantau kelengkapan assessment dan evaluation sesuai jadwal, mencatat isu operasional beserta tindak lanjut yang tidak mengubah fitur, serta menyimpan bukti komunikasi dan catatan operasional harian.

## Kesalahan Umum yang Harus Dihindari
Kesalahan yang sering terjadi adalah menetapkan role secara global, membiarkan enrollment tercatat pada event yang tidak tepat, atau menganggap evaluation menentukan kelulusan otomatis. Admin Event juga harus menghindari penutupan event sebelum assessment, evaluation, dan keputusan kelulusan lengkap, serta dilarang melakukan perubahan data lintas event atau lintas provinsi. Setiap perubahan di luar mekanisme change control saat change-freeze aktif merupakan pelanggaran tata kelola.

## Hubungan Admin Event dengan Pelatih dan Observer
Admin Event berkoordinasi langsung dengan Pelatih dan Observer terkait jadwal assessment, kelengkapan data, serta konfirmasi sesi yang berjalan. Pelatih dan Observer menyampaikan status penilaian dan catatan proses kepada Admin Event, sedangkan Admin Event memastikan hasilnya terdokumentasi untuk rapat kelulusan dan arsip operasional.
